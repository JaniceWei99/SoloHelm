| ------------ | --------------------------------------------------------------------------------------------- |

| 版本         | v1.0 — 功能初版已实现                                                                         |
| 技术栈       | Node.js / Express + 纯 HTML/CSS/JS（无框架、无构建）                                         |
| 代码规模     | ~982 行（app.js 461 + style.css 147 + index.html 173 + ideas.html 109 + server.js 92）       |
| 数据存储     | 服务端 JSON 文件（`data/tasks.json`）+ 浏览器 localStorage 回退                              |
| 页面         | 2 个：任务看板（index.html）、灵感池（ideas.html）                                           |
| OpenSpec状态 | `task-board-v1` 4/4 artifacts 完成；另有 3 个改进 change 已创建 proposal                     |

**已实现的核心功能：**
- 5 阶段任务生命周期：backlog → todo → dev → done → publish
- 拖拽 Kanban 看板（4 列）+ 列表视图切换
- 灵感池独立页面，实时搜索，语音输入（Web Speech API）
- 双向转换：灵感 ↔ 任务（"To Todo" / "To Idea"）
- 共享 Modal 弹窗 CRUD，状态标记循环（空→进行中→阻塞）
- Hero 指标面板 + Dashboard Summary（今日/风险/活跃）
- 深色玻璃拟态主题，响应式布局
- API 优先 + localStorage 离线回退
- 首次加载自动 seed 演示数据

### 优势分析 (vs 市场产品)

#### 1. 灵感到发布的完整管道（独创）

- 5 阶段生命周期 `backlog → todo → dev → done → publish` 比 Trello/Todoist 的简单状态更贴合独立开发者的实际工作流
- "Ideas Pool + Task Board" 双页面分离了**发散思维**和**执行跟踪**，Trello 没有这种原生区分
- "publish" 状态超越了普通的 "done"，对需要发布/上线的一人公司来说很实用

#### 2. 零成本自托管，完整数据所有权

- JSON 文件存储，git-friendly，可以直接版本控制
- 不依赖任何第三方 SaaS，没有订阅费用和数据锁定
- 对比 Linear ($8/月)、Notion (团队付费) 来说，长期成本为零

#### 3. 极简架构，极低运维负担

- 全栈 ~1000 行代码，没有数据库、没有构建工具、没有 Docker
- `npm start` 一键启动，对比 Jira 的部署复杂度几乎可忽略
- 不需要学习框架，任何会 HTML/JS 的人都能改

#### 4. 离线韧性

- API 不通时自动回退 localStorage，不会中断工作
- 对比 Trello 离线时基本不可用，这是实际的生产力保障

#### 5. 语音速记灵感

- Web Speech API 直接录入，不需要切换 app
- 对比 Todoist/Things 需要手动输入，语音捕捉灵感的摩擦力更低

#### 6. 拖拽 + 标记的轻量交互

- Kanban 拖拽跨列 + 标记循环 (空→进行中→阻塞) 的交互密度适中
- 对比 Linear 的复杂工作流、Jira 的配置地狱，更适合一个人决策

---

### 不足分析 (vs 市场产品)

#### 1. 代码层面的技术债

| 问题                                     | 影响                               | 严重度 |
| ---------------------------------------- | ---------------------------------- | ------ |
| `_marker` 状态仅在运行时，刷新丢失      | 标记"进行中/阻塞"后刷新页面全部消失 | 高     |
| API 调用 fire-and-forget，无错误反馈     | 用户不知道保存是否成功             | 高     |
| 搜索无 debounce，每次按键全量重渲染      | 数据多时卡顿                       | 中     |
| 无数据校验 (前后端都没有)                | 脏数据可写入                       | 中     |
| 删除操作无回滚                           | API 失败时前后端数据不一致         | 中     |

#### 2. 功能缺失对比

| 缺失功能              | Trello | Linear | Todoist | 对一人公司的影响       |
| --------------------- | ------ | ------ | ------- | ---------------------- |
| 子任务/Checklist      | 有     | 有     | 有      | 无法拆解大任务         |
| 提醒/到期日通知       | 有     | 有     | 有      | 容易遗忘 deadline      |
| 数据导出 (CSV/JSON)   | 有     | 有     | 有      | 数据迁移困难           |
| 键盘快捷键            | 有     | 强     | 有      | 效率受限               |
| 时间线/甘特视图       | 插件   | 有     | 无      | 无法做排期规划         |
| 周期性任务            | 插件   | 无     | 有      | 重复工作需手动建       |
| 数据可视化 (燃尽图等) | 插件   | 有     | 无      | 无法看趋势和效率       |
| 移动端适配/PWA        | 有     | 有     | 有      | 手机上体验差           |
| 暗/亮主题切换         | 有     | 有     | 有      | 只有暗色，白天不友好   |
| Undo/Redo             | 无     | 有     | 无      | 误操作无法恢复         |

#### 3. 用户体验短板

- 无可访问性 (ARIA, 键盘导航, focus trap)
- 列表视图标题/描述截断无 tooltip，信息损失
- 无空状态引导（列为空时没有提示）
- 删除无 undo，只有 confirm 弹窗

### 改进提案（3 个 OpenSpec Changes）

#### P0: `fix-critical-bugs` — 修复可靠性问题

> 不修这些，工具不可信赖

| 修复项         | 问题                       | 效果                         |
| -------------- | -------------------------- | ---------------------------- |
| Marker 持久化  | 刷新页面标记丢失           | 进行中/阻塞状态正确保存     |
| API 错误反馈   | 保存失败用户不知道         | Toast 提示 + 离线模式指示   |
| 搜索 debounce  | 每按键全量重渲染           | 300ms 防抖，流畅搜索        |
| 数据校验       | 可写入脏数据               | 前后端枚举/长度校验         |
| 删除回滚       | 删除 API 失败数据不一致    | 乐观更新 + 失败恢复         |
