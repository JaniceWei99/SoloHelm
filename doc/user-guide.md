## 快速开始

### 一键安装

从 GitHub 下载 ZIP 解压后，双击对应系统的文件即可：

| 系统 | 文件 | 操作 |
|------|------|------|
| **Windows** | `install-windows.bat` | 双击运行 |
| **macOS** | `install-macos.command` | 双击运行 |
| **Linux** | `install-linux.sh` | 终端执行 `bash install-linux.sh` |

脚本会自动安装 Node.js（如未安装）、安装依赖、启动服务并打开浏览器。

> **macOS 首次双击提示"无法打开"或"无法验证开发者"**：
>
> 方法一（推荐）：**右键点击** `install-macos.command` → 选择"打开" → 点击"打开"
>
> 方法二：打开 **系统设置 → 隐私与安全性** → 底部找到被阻止的文件 → 点击"仍要打开"
>
> 方法三：打开终端执行 `xattr -d com.apple.quarantine install-macos.command`，然后双击即可

### 手动安装

```bash
npm install && npm start
```

打开 http://localhost:3000 即可使用。详细部署方式（Docker / Helm / systemd）见 [部署指南](deploy.md)。

## 灵感池 (Ideas)

灵感池是所有想法的入口。状态为 `backlog` 的条目会出现在这里。

### 核心功能

- **快速记录**：点击「新建灵感」或按 `N`，输入标题和描述即可保存
- **实时搜索**：顶部搜索框支持按标题或标签搜索，输入即刻过滤（按 `/` 聚焦）
- **语音输入**：点击描述框旁的麦克风图标，通过语音快速录入描述
- **转为任务**：灵感卡片上的「转待办」按钮可将灵感状态改为 `todo`，移入看板

### Hero 指标

| 指标 | 含义 |
|------|------|
| 灵感总数 | 灵感池中的条目数量 |
| 开发中 | 状态为 `dev` 的任务数（从灵感转化而来） |

## 看板 (Board)

看板是任务管理的核心区域，支持三种视图切换。

### 三种视图

#### 看板视图 (Kanban)

默认视图。任务按状态分为四列：

| 列名 | 对应状态 | 说明 |
|------|---------|------|
| 待办 | `todo` | 等待开始的任务 |
| 进行中 | `dev` | 正在开发中 |
| 开发完成 | `done` | 开发完毕，待发布 |
| 已发布 | `publish` | 已上线 |

每列顶部显示任务数量，支持拖拽移动任务到其他列。

#### 列表视图 (List)

紧凑的表格视图，显示所有非 backlog 状态的任务，包含类型、标题、描述、标签、优先级、发布时间、状态和操作列。

#### 时间线视图 (Timeline)

按发布日期在时间轴上展示任务，只有设置了发布日期的任务才会出现。今日日期会高亮标注。

### Hero 指标面板

看板顶部的统计卡片：

| 指标 | 含义 |
|------|------|
| 进行中 | `todo` + `dev` 状态的任务总数 |
| 已完成 | `done` 状态的任务数 |
| 灵感池 | `backlog` 状态的数量 |
| 已发布 | `publish` 状态的数量 |

### 仪表板摘要

| 指标 | 含义 |
|------|------|
| 今日 | 今天完成或发布的任务数 |
| 风险/阻塞 | 标记为风险的任务数 |
| 活跃 | 进行中的任务数（todo + dev） |

## 分析 (Analytics)

数据可视化面板，自动从后端 `/api/analytics` 获取数据。

### Hero 指标

| 指标 | 含义 |
|------|------|
| 总任务数 | 所有任务总量 |
| 平均周期 (天) | 从创建到完成/发布的平均天数 |
| 本周 | 本周完成的任务数量 |

### 四张图表

| 图表 | 类型 | 说明 |
|------|------|------|
| 状态分布 | 环形图 (Doughnut) | 各状态的任务占比 |
| 优先级分布 | 柱状图 (Bar) | P1/P2/P3 各有多少任务 |
| 每周速率 | 折线图 (Line) | 每周完成任务的数量趋势 |
| 燃尽图 | 折线图 (Line) | 累计创建 vs 累计完成的对比 |

> 切换主题或语言后图表会自动重新渲染以匹配配色。

## 创建与编辑任务

### 打开新建弹窗

- 点击顶部工具栏的 **「+ 新建任务」** 按钮
- 或按快捷键 `N`
- 在灵感 Tab 时按钮显示为「新建灵感」，在看板 Tab 时显示为「新建任务」

### 表单字段

| 字段 | 必填 | 说明 | 限制 |
|------|------|------|------|
| 标题 | 是 | 任务/灵感的名称 | 最长 200 字符 |
| 描述 | 否 | 详细说明，支持语音输入 | 最长 2000 字符 |
| 标签 | 否 | 逗号分隔的标签 | 最多 20 个，每个最长 50 字符 |
| 优先级 | 否 | P1 (高) / P2 (中) / P3 (低) | 默认 P3 |
| 类型 | 否 | 功能 / 缺陷 / 灵感 | 默认：看板=功能，灵感池=灵感 |
| 状态 | 否 | 待定 / 待办 / 开发 / 完成 / 发布 | 默认：看板=待办，灵感池=待定 |
| 依赖 | 否 | 依赖的任务或条件 | 最长 200 字符 |
| 发布时间 | 否 | 预计发布日期 | 日期选择器 |
| 重复 | 否 | 无 / 每日 / 每周 / 每月 | 设置后完成时自动克隆 |
| 子任务 | 否 | 可勾选的清单项 | 最多 50 个 |

### 编辑

点击任务卡片的「编辑」按钮，弹窗会预填当前数据，修改后保存即可。

### 前后端双重校验

前端在提交前检查必填项和长度限制，后端同样验证。无效数据会以 Toast 提示具体错误。

## 子任务 / 清单

任务可以包含子任务（Checklist），用于拆分细粒度的步骤。

### 使用方法

1. 在新建/编辑弹窗底部的「子任务/清单」区域
2. 输入子任务文字，点击 `+` 或按 `Enter` 添加
3. 勾选复选框标记完成，点击 `×` 删除

### 进度条

任务卡片上会显示进度条和完成比例（如 `2/5`），一目了然。

## 周期性任务

适用于每日站会、每周总结等重复性任务。

### 设置方式

在新建/编辑弹窗中选择「重复」频率：

| 选项 | 效果 |
|------|------|
| 无 | 普通任务，无重复 |
| 每日 | 完成后自动克隆为新的 todo |
| 每周 | 同上 |
| 每月 | 同上 |

### 工作原理

当周期性任务移动到 `done` 或 `publish` 状态时，系统会自动通过 `/api/tasks/:id/clone` 克隆一份新任务：
- 新任务状态为 `todo`
- 子任务重置为未完成
- 保留原始标题、描述、标签、优先级等
- 会弹出 Toast 提示「重复任务已克隆」

## 状态标记 (Marker)

卡片左上角的标记图标，用于快速标注任务的工作状态。

### 点击循环

点击标记图标会在三种状态间循环：

```
无标记 → 进行中 (▶) → 已阻塞 (⚠) → 无标记
```

| 标记 | 图标 | 说明 |
|------|------|------|
| (无) | ● (灰色) | 默认状态 |
| 进行中 | ▶ (绿色) | 正在处理 |
| 已阻塞 | ⚠ (红色) | 遇到阻碍，同时标记 risk=true |

> 标记为「已阻塞」的任务会计入仪表板的「风险/阻塞」指标。

## 拖拽操作

在看板视图中，可以通过拖拽将任务卡片移动到其他列，自动更新状态。

### 操作步骤

1. 鼠标按住任务卡片
2. 拖动到目标列
3. 松开鼠标

拖入的列决定新状态（todo / dev / done / publish）。状态变更支持撤销（Ctrl+Z）。

## 搜索

灵感池 Tab 提供实时搜索功能。

### 使用方法

1. 切换到灵感 Tab
2. 点击搜索框或按 `/` 聚焦
3. 输入关键词，300ms 防抖后自动过滤

### 搜索范围

- 标题（模糊匹配）
- 标签（模糊匹配）

## 任务详情面板

点击任务卡片的标题，右侧会滑出详情面板，展示完整信息。

### 显示内容

- 描述、状态、优先级、类型

## 变更历史

所有操作都会被记录，支持两种查看方式。

### 任务历史

点击任务卡片上的时钟图标，右侧面板显示该任务的全部变更记录：
- 创建、更新、删除事件
- 每次更新会记录具体字段的变化（旧值 → 新值）

### 全局历史

点击顶部工具栏的时钟图标，查看最近 50 条全局活动：
- 显示任务名称 + 操作类型
- 按时间倒序排列

## 数据导出与导入

通过设置菜单（齿轮图标）操作。

### 导出

| 格式 | 说明 |
|------|------|
| JSON | 完整数据，可用于备份和导入恢复（服务端 API） |
| CSV | 表格格式，可在 Excel/Google Sheets 中打开（浏览器端生成） |

导出的文件命名格式：`solohelm-YYYY-MM-DD.json` / `.csv`

> CSV 导出在浏览器端生成并直接下载，不经过服务端 API。JSON 导出通过 `/api/export` 获取。

### 导入

1. 点击「导入 JSON」
2. 选择之前导出的 `.json` 文件
3. 系统校验每条数据，跳过无效项和重复 ID
4. Toast 提示导入结果（如「已导入 8/10 个任务」）

> 单次导入上限 500 条任务。

## 主题切换

### 操作

点击顶部工具栏的太阳/月亮图标切换深色和浅色主题。

### 行为

- 首次访问时跟随系统偏好（`prefers-color-scheme`）
- 选择后保存到 `localStorage`，下次打开自动恢复
- **侧边栏始终保持深色**，仅内容区跟随主题切换
- 切换主题后分析图表会自动重新渲染

## 中英文切换

### 操作

点击顶部工具栏的地球图标，在中文和英文之间切换。

### 行为

- 默认中文
- 覆盖 ~150 个翻译 key，包括导航、按钮、表单、提示、图表标签等
- 选择后保存到 `localStorage`
- 切换语言后所有界面文本实时更新，无需刷新

## 语音输入

在新建/编辑弹窗中，描述字段旁有麦克风图标。

### 使用方法

1. 点击麦克风图标开始录音（图标高亮）
2. 对着麦克风说话
3. 再次点击或说完后自动停止
4. 语音识别的文本会追加到描述框中

### 注意事项

- 依赖 Web Speech API，需要浏览器支持（Chrome/Edge 支持较好）
- 默认识别语言为中文
- 不支持的浏览器会显示提示

## 撤销操作

删除任务和状态变更支持 5 秒内撤销。

### 工作方式

1. 执行删除或状态变更后，底部弹出带「撤销」按钮的 Toast
2. 点击「撤销」或按 `Ctrl+Z` 可恢复
3. 5 秒后 Toast 消失，操作不可撤销

### 支持的撤销

| 操作 | 撤销效果 |
|------|---------|
| 删除任务 | 通过 API 重新创建任务，恢复所有数据 |
| 状态变更 | 恢复到变更前的状态 |

## PWA 离线支持

SoloHelm 是一个 PWA (Progressive Web App)，支持离线使用和安装。

### 安装

- **桌面**：Chrome 地址栏右侧会出现安装图标，点击即可安装到桌面
- **手机**：Safari/Chrome 中选择「添加到主屏幕」

### 离线模式

- Service Worker 会缓存页面资源
- 离线时使用 `localStorage` 中的本地数据
- 界面会弹出「离线模式 — 使用本地数据」提示
- 恢复网络后刷新页面即可同步

## 任务生命周期

SoloHelm 使用 5 阶段生命周期管理任务，从灵感到发布：

```
backlog (灵感池)
   │
   ▼  "转待办"
  todo (待办)
   │
   ▼  拖拽到进行中
  dev (开发)
   │
   ▼  拖拽到开发完成
  done (完成)
   │
   ▼  拖拽到已发布
publish (发布)
```

### 状态说明

| 状态 | 中文 | 所在区域 | 说明 |
|------|------|---------|------|
| `backlog` | 待定 | 灵感池 | 初始想法，尚未进入开发流程 |
| `todo` | 待办 | 看板第 1 列 | 确认要做的任务 |
| `dev` | 开发 | 看板第 2 列 | 正在开发中 |
| `done` | 完成 | 看板第 3 列 | 开发完毕，待发布 |
| `publish` | 发布 | 看板第 4 列 | 已上线发布 |

### 灵感 ↔ 任务流转

- **灵感转任务**：在灵感池点击「转待办」，状态变为 `todo`
- **任务退回灵感**：在看板点击「转灵感」，状态变为 `backlog`，类型变为 `idea`

### 状态变更方式

1. **拖拽**：看板视图中拖动卡片到目标列
2. **按钮**：「转待办」/「转灵感」快捷按钮
3. **编辑**：打开编辑弹窗手动选择状态
